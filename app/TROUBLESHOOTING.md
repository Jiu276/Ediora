# API 调用失败故障排查指南

## 🔴 问题：fetch failed / 连接中断

### 可能原因

1. **网络连接问题**（最常见）
   - 国内访问 Google API 可能被墙
   - 网络不稳定
   - 防火墙阻止

2. **API Key 问题**
   - API Key 无效或过期
   - API Key 权限不足

3. **配额限制**
   - 超出免费额度
   - 速率限制（每分钟15次）

## ✅ 解决方案

### 方案一：使用模板生成（推荐，立即可用）

如果 AI API 无法访问，可以使用模板生成作为备选方案：

1. 在标题生成页面
2. 点击 **"模板生成 (备选)"** 按钮
3. 系统会使用模板生成10个标题
4. 虽然不是 AI 生成，但可以正常使用

### 方案二：使用代理/VPN

如果需要使用 AI 生成，可以：

1. **配置系统代理**
   - 使用 VPN 或代理软件
   - 确保可以访问 `generativelanguage.googleapis.com`

2. **配置 Node.js 代理**（服务器端）
   在 `.env.local` 中添加：
   ```
   HTTP_PROXY=http://127.0.0.1:7890
   HTTPS_PROXY=http://127.0.0.1:7890
   ```
   注意：需要重启开发服务器

### 方案三：切换到其他 AI 服务

如果 Google API 持续无法访问，可以考虑：

1. **DeepSeek**（推荐）
   - 国内访问稳定
   - 价格便宜（约 0.2 分/次）
   - 需要修改 `.env.local` 配置

2. **通义千问**（阿里云）
   - 国内访问稳定
   - 有免费额度

## 🔍 详细排查步骤

### 1. 检查网络连接

在浏览器控制台运行：
```javascript
fetch('https://generativelanguage.googleapis.com')
  .then(() => console.log('可以访问'))
  .catch(() => console.log('无法访问'))
```

### 2. 检查 API Key

1. 访问：https://aistudio.google.com/
2. 登录账号
3. 检查 API Key 是否有效
4. 查看使用量和配额

### 3. 查看详细错误

打开浏览器开发者工具（F12）：
- 查看 **Console** 标签的错误信息
- 查看 **Network** 标签的请求详情
- 检查请求状态码和响应内容

### 4. 检查服务器日志

查看终端/控制台的错误信息：
- 是否有网络错误
- 是否有 API 错误
- 错误详情是什么

## 📋 常见错误码

| 错误码 | 含义 | 解决方案 |
|--------|------|----------|
| 401 | API Key 无效 | 检查 API Key 是否正确 |
| 403 | 访问被拒绝 | 检查配额和权限 |
| 429 | 请求过于频繁 | 等待后重试 |
| 500+ | 服务器错误 | 稍后重试 |
| fetch failed | 网络错误 | 使用代理或模板生成 |

## 🛠️ 临时解决方案

如果急需使用，可以：

1. **使用模板生成**
   - 点击"模板生成 (备选)"按钮
   - 虽然不是 AI，但可以正常使用

2. **手动输入标题**
   - 直接在"已保存的标题"页面添加
   - 或使用文章管理页面的标题功能

2. 查看服务器终端的错误日志
3. 提供错误截图和详细信息
