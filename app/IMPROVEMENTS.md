# Ediora 系统完善建议

> 本文档提供系统完善的方向和建议，不包含具体实现代码

## 📊 当前已实现功能概览

### ✅ 已完成的核心功能
- 文章 CRUD（创建、读取、更新、删除）
- 发布向导（4步流程）
- 标签类别管理
- 文章类型管理
- 前端文章展示（列表页、详情页）
- 数据库集成（MySQL + Prisma）
- 软删除机制

---

## 🎯 完善建议（按优先级分类）

### 🔴 P0 - 核心功能完善（必须实现）

#### 1. 数据关联完整性
**问题**：
- 发布向导中生成的 `customDomains`、`links`、`images` 数据未保存到关联表
- 文章与标签类别、文章类型的关联关系未在前端展示

**建议**：
- 实现 `ArticleCustomDomain`、`ArticleLink`、`ArticleImage` 的 CRUD API
- 在发布向导完成后，将数据保存到对应的关联表
- 在文章详情页展示关联的域名、超链接、配图
- 在文章列表页显示标签类别信息

**影响**：数据完整性、功能可用性

---

#### 2. 文章标签系统
**问题**：
- 当前只有标签类别（Category），缺少文章标签（Tag）功能
- 无法为文章添加多个标签

**建议**：
- 实现 `ArticleTag` 的 CRUD API
- 在文章编辑页面添加标签选择/输入功能
- 在前端展示文章标签
- 实现标签云/标签筛选功能

**影响**：内容分类、SEO、用户体验

---

#### 3. 富文本编辑器
**问题**：
- 当前使用普通文本域，无法格式化内容
- 无法插入图片、链接等富媒体

**建议**：
- 集成 TipTap 或 Quill 富文本编辑器
- 支持标题、加粗、斜体、列表等基础格式
- 支持图片上传和插入
- 支持代码块、引用等高级格式
- 保存 HTML 格式内容

**影响**：内容创作体验、内容质量

---

#### 4. 图片上传功能
**问题**：
- 当前只能输入图片 URL，无法上传本地图片
- 没有图片管理功能

**建议**：
- 实现图片上传 API（支持本地存储或云存储）
- 创建图片管理页面（Asset 管理）
- 在富文本编辑器中集成图片上传
- 支持图片裁剪、压缩
- 实现图片库功能

**影响**：内容创作效率、用户体验

---

### 🟡 P1 - 重要功能增强（强烈建议）

#### 5. 搜索和筛选功能
**建议**：
- 在文章列表页添加搜索框（按标题、内容搜索）
- 添加筛选器（按状态、标签类别、文章类型、日期范围）
- 实现全文搜索（如使用 MySQL FULLTEXT 或 Elasticsearch）
- 添加排序功能（按日期、标题、作者）

**影响**：内容管理效率

---

#### 6. 批量操作
**建议**：
- 文章列表支持多选
- 批量删除、批量发布、批量修改状态
- 批量修改标签类别、文章类型
- 批量导出功能

**影响**：管理效率

---

#### 7. 文章预览功能
**建议**：
- 在编辑页面添加"预览"按钮
- 实时预览文章在前端的展示效果
- 支持移动端预览模式

**影响**：内容质量保障

---

#### 8. 定时发布功能
**建议**：
- 实现定时任务系统（使用 node-cron 或类似工具）
- 检查 `publishDate` 字段，自动发布到期文章
- 在文章列表显示定时发布状态
- 支持修改定时发布时间

**影响**：内容运营效率

---

#### 9. 关键字自动超链接实现
**问题**：
- 当前只有开关，未实现实际功能

**建议**：
- 在文章内容渲染时，自动将关键字替换为超链接
- 支持从 `ArticleLink` 表读取关键字和 URL
- 避免重复替换（已存在的链接不处理）
- 支持配置替换规则（大小写敏感、全词匹配等）

**影响**：SEO、用户体验

---

#### 10. 文章版本历史
**建议**：
- 创建文章版本表（ArticleVersion）
- 每次保存时创建版本快照
- 支持查看历史版本
- 支持恢复到历史版本
- 显示版本对比（diff）

**影响**：内容安全、协作效率

## 📋 技术债务和优化建议

### 代码质量
1. **错误处理**：统一错误处理机制，添加错误边界
2. **类型安全**：完善 TypeScript 类型定义
3. **代码复用**：提取公共组件和工具函数
4. **测试**：添加单元测试和集成测试
5. **文档**：完善 API 文档和代码注释

### 安全性
1. **输入验证**：所有用户输入进行验证和清理
2. **SQL 注入防护**：确保 Prisma 查询安全
3. **XSS 防护**：HTML 内容渲染时进行转义
4. **CSRF 防护**：添加 CSRF token
5. **API 限流**：防止 API 滥用

### 数据库优化
1. **索引优化**：检查并优化数据库索引
2. **查询优化**：使用 Prisma 的连接查询减少 N+1 问题
3. **数据迁移**：建立规范的数据库迁移流程
4. **数据清理**：实现定期清理软删除数据的功能

## 📝 注意事项

1. **数据迁移**：在添加新功能时，注意处理现有数据
2. **向后兼容**：确保新功能不影响现有功能
3. **性能监控**：关注系统性能，及时优化
4. **用户反馈**：收集用户反馈，优先实现高频需求
5. **文档更新**：功能更新时同步更新文档
