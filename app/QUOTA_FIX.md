# 配额问题修复指南

## 🔴 问题：配额层级显示"不可用"

从你的 Google AI Studio 截图可以看到：
- 项目：test
- 配额层级：**设置结算信息** / **不可用**

这可能是导致 API 调用失败的主要原因。

## ✅ 解决方案

### 方案 1：设置结算信息（推荐）

1. **访问 Google AI Studio**
   - 打开：https://aistudio.google.com/
   - 登录你的账号

2. **进入项目设置**
   - 点击左侧菜单的"项目"（Projects）
   - 找到你的项目（如 "test"）
   - 点击项目名称进入详情

3. **设置结算信息**
   - 点击"配额层级"旁边的"设置结算信息"
   - 按照提示添加支付方式（信用卡等）
   - 即使有免费额度，也可能需要绑定支付方式

4. **等待激活**
   - 设置完成后，等待几分钟让配额生效
   - 刷新页面，确认配额状态变为"可用"

### 方案 2：使用其他模型

如果某些模型需要付费，可以尝试：

1. **gemini-pro**（旧版本，可能免费）
   ```env
   GOOGLE_AI_MODEL=gemini-pro
   GOOGLE_AI_API_VERSION=v1beta
   ```

2. **检查可用模型**
   - 在 Google AI Studio 查看哪些模型可用
   - 查看每个模型的配额要求

### 方案 3：创建新项目

如果当前项目有问题，可以：

1. **创建新项目**
   - 点击"+ 创建新项目"
   - 输入项目名称
   - 创建新的 API Key

2. **设置新项目**
   - 在新项目中设置结算信息
   - 获取新的 API Key
   - 更新 `.env.local` 文件

## 🔍 检查步骤

### 1. 检查项目状态

在 Google AI Studio：
- 查看项目列表
- 确认配额状态是否为"可用"
- 如果显示"不可用"，需要设置结算信息

### 2. 检查 API Key 权限

1. 点击"API 密钥"（API Keys）
2. 查看你的 API Key
3. 确认 Key 是否启用
4. 检查 Key 的权限范围

### 3. 测试 API 调用

在 Google AI Studio 的 Playground：
1. 尝试使用不同的模型
2. 查看哪些模型可用
3. 查看错误信息

## 📝 更新配置

根据你的项目情况，更新 `.env.local`：

```env
# 如果配额不可用，先尝试 gemini-pro（旧版本）
GOOGLE_AI_MODEL=gemini-pro
GOOGLE_AI_API_VERSION=v1beta

# 或者如果设置了结算信息，可以使用新模型
GOOGLE_AI_MODEL=gemini-1.5-flash
GOOGLE_AI_API_VERSION=v1
```

## ⚠️ 重要提示

1. **免费额度**
   - Google AI Studio 有免费额度
   - 但可能需要绑定支付方式才能激活
   - 绑定后不会立即扣费，只在超出免费额度时收费

2. **配额层级**
   - "不可用" = 无法使用 API
   - "免费层级" = 可以使用免费额度
   - "付费层级" = 可以使用付费功能

3. **模型可用性**
   - 不同模型可能有不同的配额要求
   - 某些新模型可能需要付费层级
   - 旧模型（如 gemini-pro）可能免费可用

## 🚀 快速修复步骤

1. **设置结算信息**
   - 在 Google AI Studio 项目页面
   - 点击"设置结算信息"
   - 添加支付方式

2. **等待激活**
   - 等待 5-10 分钟
   - 刷新页面确认状态

3. **更新配置**
   - 如果还是不行，尝试使用 `gemini-pro` 模型
   - 更新 `.env.local` 文件
   - 重启服务器

4. **测试**
   - 在标题生成页面测试
   - 查看错误信息
   - 根据错误调整配置
