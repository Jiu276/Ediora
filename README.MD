# Ediora · 博客文章发布管理系统

> 目标：帮助内容创作者高效管理从灵感到上线的整个内容生命周期，确保每一次发布都能按计划、高质量地抵达目标读者。

---

## 1. 项目背景与价值

- **痛点聚焦**：创作者缺少统一平台追踪创作进度、协同反馈、排期上线与质量复盘，导致内容延迟或质量波动。
- **目标用户**：独立内容创作者、内容工作室、小型自媒体团队。
- **价值主张**：以可视化内容日历、规范化质检流程、自动化分发清单为核心，提供端到端的内容运营支撑。

---

## 2. Demo 功能需求文档

> 本文档描述当前 Demo（2025-12-03）的真实可用功能，作为后续迭代的基线需求。

### 2.1 管理后台

| 模块 | 能力 | 说明 |
| --- | --- | --- |
| 文章列表 | 查看 / 编辑 / 删除 / 状态切换 | 展示标题、状态、作者、发布时间、创建时间 |
| 文章编辑 | 草稿编辑、保存草稿、标签管理 | 支持标题、摘要、正文、状态、发布日期、标签 |
| 发布向导 | 选择标签类别 → 选择标题 → 添加超链接 → 生成文章和配图 | 智能生成标题、内容、配图，支持自定义域名、超链接、封面、多图 |
| 标签类别管理 | CRUD | 菜单项「标签类别」，可维护生活 / 旅游 / 科技等分类 |
| 文章类型管理 | CRUD | 原有功能保留 |
| 主题管理 | 激活主题 | 影响前台展示样式（占位功能） |

### 2.2 发布向导细节
1. **选择标签类别**
   - 下拉选择类别（来源于标签类别管理）
   - 支持添加多个自定义域名（用于生成标题 / 内容）
2. **选择标题**
   - 根据标签类别 + 域名生成 10 个标题建议，可直接点击选择
   - 支持输入自定义标题
3. **添加超链接**
   - 可添加多个关键字 + URL 组合
   - 支持删除、校验 URL
4. **生成文章和配图**
   - 根据标题自动生成分段文章（Demo 使用模拟数据）
   - 自动生成 5 张配图，要求用户选择 3-5 张
   - 设置发布日期、启用/关闭关键字自动超链接

发布后写入 `data.json`，包含以下字段：`categoryId`, `customDomains`, `links`, `images`, `featuredImage`, `enableKeywordLinks`, `publishDate` 等。

### 2.3 前台展示站
- 路由：`/`、`/blog`（文章列表）、`/blog/:slug`（详情）
- 文章列表展示封面、标题、摘要、发布日期、标签，点击跳转详情
- 详情页展示封面、标题、作者、发布时间、正文、标签
- 所有页面响应式设计，适配桌面和移动端

### 2.4 数据流 & 自动化
1. 标签类别 → 发布向导选择
2. 生成标题 → 10 条推荐标题，带随机推荐度
3. 生成文章内容 → 按类别输出不同段落结构
4. 自动配图 → 根据标题关键词生成 3-5 张示例图
5. 超链接 → 手动添加的关键词与 URL 随文章保存
6. 发布 → 后台写入 JSON，前台实时读取

### 2.5 验证要点
- 发布向导 4 步流程可完整走通
- 标签类别管理的数据实时应用
- 新文章发布后可在 `/blog` / `/blog/:slug` 访问
- 自动生成的内容（标题 / 正文 / 配图）可成功保存与展示
- API 返回 200 且无异常

> 注：更早期 README 中描述的“创意池 / 排期 / 协作”等高级特性尚未实现，保留为愿景规划。

## 5. 技术框架详细说明

### 5.1 前端技术栈

#### 5.1.1 React 18.x
- **作用**：构建用户界面的 JavaScript 库
- **用途**：
  - 组件化开发，提高代码复用性
  - 声明式 UI，简化状态管理
  - 虚拟 DOM，优化渲染性能
- **版本选择原因**：稳定、生态成熟、支持 Hooks

#### 5.1.2 Vite 5.x
- **作用**：下一代前端构建工具
- **用途**：
  - 快速开发服务器（HMR 热更新）
  - 生产环境构建优化
  - 支持 ES 模块
- **配置要点**：
  - 开发服务器端口：前端 3000，管理后台 3002
  - API 代理：`/api` → `http://localhost:3001/api`
  - 构建输出：`dist/` 目录

#### 5.1.3 React Router 6.x
- **作用**：React 应用的路由管理库
- **用途**：
  - 前端路由配置（无需后端路由）
  - 页面跳转和参数传递
  - 路由守卫（后续扩展认证）
- **路由模式**：BrowserRouter（使用 HTML5 History API）

#### 5.1.4 CSS
- **作用**：样式定义
- **使用方式**：
  - 组件级 CSS 文件（如 `Home.css`）
  - 全局样式文件（`index.css`、`App.css`）
  - 内联样式（少量动态样式）
- **设计原则**：
  - 响应式设计（移动端优先）
  - BEM 命名规范（可选）
  - CSS 变量（颜色、间距等）

### 5.2 后端技术栈

#### 5.2.1 Node.js 18+
- **作用**：JavaScript 运行时环境
- **用途**：运行 Express 服务器
- **版本要求**：支持 ES6+ 语法

#### 5.2.2 Express 4.x
- **作用**：Node.js Web 应用框架
- **用途**：
  - RESTful API 路由定义
  - 中间件配置（CORS、Body Parser）
  - 静态文件服务
- **核心功能**：
  - `app.get()` - GET 请求处理
  - `app.post()` - POST 请求处理
  - `app.put()` - PUT 请求处理
  - `app.delete()` - DELETE 请求处理

#### 5.2.3 CORS
- **作用**：跨域资源共享中间件
- **用途**：允许前端（3000、3002 端口）访问后端 API（3001 端口）
- **配置**：`app.use(cors())` - 允许所有来源（开发环境）

#### 5.2.4 Body Parser
- **作用**：解析 HTTP 请求体
- **用途**：解析 JSON 格式的请求数据
- **配置**：`app.use(bodyParser.json())`

#### 5.2.5 UUID
- **作用**：生成唯一标识符
- **用途**：为文章、类型、类别等实体生成唯一 ID
- **使用**：`uuidv4()` 生成随机 UUID

## 6. API 接口文档

### 6.1 文章管理

- `GET /api/posts` - 获取所有文章（支持 `?status=published` 过滤）
- `GET /api/posts/:id` - 根据 ID 获取文章
- `GET /api/posts/slug/:slug` - 根据 slug 获取文章（前台用）
- `POST /api/posts` - 创建文章
- `PUT /api/posts/:id` - 更新文章
- `DELETE /api/posts/:id` - 删除文章

### 6.2 文章类型管理

- `GET /api/content-types` - 获取所有内容类型
- `GET /api/content-types/:id` - 获取单个内容类型
- `POST /api/content-types` - 创建内容类型
- `PUT /api/content-types/:id` - 更新内容类型
- `DELETE /api/content-types/:id` - 删除内容类型

### 6.3 主题管理

- `GET /api/themes` - 获取所有主题
- `GET /api/themes/active` - 获取当前激活的主题
- `PUT /api/themes/:id/activate` - 激活指定主题

### 6.4 域名管理

- `GET /api/domains` - 获取所有域名
- `GET /api/domains/:id` - 获取单个域名
- `POST /api/domains` - 创建域名
- `PUT /api/domains/:id` - 更新域名
- `DELETE /api/domains/:id` - 删除域名

### 6.5 标题生成

- `POST /api/generate-titles` - 根据内容和类型生成10个标题建议
  - 请求体：`{ content, typeId, domainIds }`
  - 返回：`{ titles: [{ id, title, score }] }`

### 6.6 自动配图

- `POST /api/auto-images` - 根据文章内容自动获取配图建议
  - 请求体：`{ content, title }`
  - 返回：`{ images: [{ id, url, thumbnail, description, source }], searchTerm }`

### 6.7 资源管理

- `GET /api/assets` - 获取所有资源
- `POST /api/assets` - 上传资源

## 7. 文章发布流程

### 7.1 完整发布流程（向导式）

1. **创建文章内容**
   - 访问管理后台 → 「文章」→「新建文章」
   - 填写文章标题、内容、摘要等基本信息
   - 保存为草稿

2. **启动发布向导**
   - 在文章编辑页面点击「发布文章」按钮
   - 进入三步发布向导

3. **步骤 1：选择类型和域名**
   - 选择文章类型（必选）
   - 选择相关域名（可多选）
   - 点击「下一步」

4. **步骤 2：选择标题**
   - 系统自动生成10个标题建议（基于内容、类型、域名）
   - 选择其中一个标题，或自定义标题
   - 点击「下一步」

5. **步骤 3：配图和设置**
   - 系统根据文章内容自动推荐配图（来自 Unsplash）
   - 选择一张配图作为封面
   - 设置发布日期
   - 选择是否启用关键字自动超链接
   - 点击「发布」完成

### 7.2 文章管理功能

1. **文章增删查改**
   - 列表查看所有文章
   - 新建、编辑、删除文章
   - 修改文章状态（草稿/已发布）

2. **发布日期管理**
   - 在编辑页面可修改发布日期
   - 文章列表显示发布日期
   - 支持定时发布（通过发布日期控制）

3. **文章类型管理**
   - 创建、编辑、删除文章类型
   - 为文章分配类型

4. **前端样式选择**
   - 在「主题」页面选择并激活不同主题
   - 主题会影响前端展示站的样式

### 7.3 前端展示功能

1. **首页：文章列表展示**
   - 显示所有已发布的文章
   - 支持按时间排序
   - 显示文章标题、摘要、发布日期、标签
   - 响应式设计，适配移动端

2. **文章详情页：完整内容展示**
   - 显示完整文章内容
   - 显示作者、发布日期、标签等信息
   - 支持 HTML 内容渲染
   - 响应式设计

3. **响应式设计**
   - 所有页面均支持移动端、平板、桌面端
   - 自适应布局和字体大小
   - 优化的触摸交互
